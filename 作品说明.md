# AI 新闻中心（ai-news-hub）· 作品说明

> 本文档面向评委，说明作品的创作思路、应用场景、功能、技术架构与创新点，帮助快速理解项目价值与完成度。

## 0. 项目速览（30 秒看懂）

- **是什么**：一个**中英双语 AI 资讯聚合平台**，将全球数十个 AI 信源聚合成一站，并扩展出产业情报（私募 / 传媒）与前沿论文追踪。
- **技术栈**：Next.js 14（App Router）+ TypeScript + Tailwind CSS，部署于 Vercel，数据层使用 Upstash Redis。
- **目标用户**：中国 AI 从业者、研究者、科技爱好者，以及关注 AI 行业动态的专业 / 普通用户。
- **核心差异**：不只是"搬运 RSS"，而是形成 **自动翻译 → 全文抓取 → 结构化情报 → 邮件摘要** 的内容闭环。

## 一、创作思路

**1. 痛点驱动**
AI 信息高度分散：中文有量子位、机器之心、36氪，英文有 TechCrunch、MIT Tech Review、OpenAI Blog 等数十个源；英文源对中文用户存在语言壁垒，且信息过载、追踪成本高。

**2. 设计哲学**
- **专业可信**：不做成"一眼 AI 生成"的模板站，而像一个值得信赖的 AI 媒体。
- **有温度、有活力**：现代简洁、留白、有节奏感，而非冰冷工具。
- **刻意反套路**：避开 32px+ 过度圆角、避开"AI 通用感"的视觉语言。
- **双轨内容观**：既提供"实时资讯流"（快），也提供"结构化产业情报"（深）——私募 / 创投动态、传媒声量监测，并带优先级与可信度标注。

## 二、应用场景

| 场景 | 说明 |
|---|---|
| 每日 AI 速览 | 首页即看当日中英双语精选，按时间排序 |
| 跨语言深度学习 | 英文源自动译中，降低阅读门槛 |
| 产业情报监测 | 私募 / 创投 GP 动态、传媒声量，带优先级 / 可信度矩阵 |
| 前沿论文追踪 | 聚合 arXiv / OpenAlex / Semantic Scholar |
| 邮件订阅摘要 | 用户可选择 AI / 传媒 / 私募或全部，通过 Buttondown 标签分组推送 |
| 个人知识管理 | 收藏夹 + 阅读历史（本地优先，零账号） |

## 三、作品功能

**资讯核心**
- 中英 RSS 聚合（13 个源），自动分类归一化。
- 全文抓取：RSS 摘要过短时，用 cheerio 抓取原文正文。
- 自动翻译：英文源经 Google Translate 翻成中文，标题 / 摘要 / 正文分别处理并保留 HTML 结构，含 24h 缓存。
- 分类浏览、专题导航、趋势热门区、相关文章、面包屑、分页、分享。

**情报系统（独立模块）**
- 私募 / 创投情报、传媒监测，数据源归一化为 `IntelArticle`（含 company、priority、dimension、matrixLabel、credibility 等结构化字段），带优先级矩阵与可信度评级。

**论文频道**：聚合 arXiv / OpenAlex / Semantic Scholar。

**个人化与互动**：收藏夹、阅读历史（localStorage，无需账号）、客户端全文搜索（flexsearch 内存索引）、三栏目邮件偏好与摘要预览。

**体验与合规**：暗色 / 浅色双主题（防闪烁）、`reduced-motion`、WCAG 2.1 AA 取向；完整 SEO（`sitemap` / `robots` / OpenGraph）、`/rss.xml` 对外输出。

## 四、技术架构

```mermaid
graph TD
  A[13个 RSS 信源 中英] --> B[fetcher: 解析+全文抓取+翻译]
  B --> C[Upstash Redis 按月分桶/90天过期]
  C --> D[页面云端优先读取]
  D --> E[页面 + /api/*]

  F[结构化快照 + 传媒/私募 RSS] --> G[统一 IntelArticle 模型]
  G --> C
  G --> I[/private-equity · /media 页面]

  E -.flexsearch.-> S[客户端搜索]
  E -.Vercel Cron.-> N[按 AI/传媒/私募标签发送邮件]
```

- **前端**：Next.js 14 App Router + TypeScript + Tailwind，全部页面 `force-dynamic`（实时数据），设计主色 `#E85D26`。
- **新闻管线**：`rss-parser` 解析 → 短摘要则 cheerio 抓全文 → Google Translate 翻译 → Upstash Redis 按月分桶（`HSETNX` 去重、90 天过期）→ 页面云端优先读取。定时抓取与阅读请求解耦，避免首屏等待外部信源。
- **情报管线**：外部自动化 JSON、传媒 RSS 与私募主题 RSS 统一归一化为 `IntelArticle`，与 Redis 历史数据增量合并；页面从云端读取并以本地历史归档降级，不会因单次抓取量变少而覆盖旧数据。
- **横切能力**：搜索（flexsearch 客户端）、分栏目邮件（Buttondown 标签 + Vercel Cron）、订阅偏好 Redis 双写、双主题、内容净化（dompurify）。

## 五、创新点

1. **中英双语一站式聚合 + 自动翻译**：消除语言壁垒，把英文一手源变成中文可读流。
2. **"实时流 + 结构化情报"双轨制**：普通新闻是时间流，私募 / 传媒是带优先级、可信度矩阵的产业情报，满足"快看"与"深研"两种需求。
3. **轻量 Serverless 采集管线**：抓取 → 翻译 → Redis 存储 → 合并去重，全程无长期运行服务；缓存与去重兼顾性能与成本。
4. **本地优先的个人化**：收藏 / 阅读历史走 localStorage，**无需账号体系**即可使用，降低门槛。
5. **刻意反"AI 模板感"的设计**：以"有温度的专业媒体"为气质，避开通用 AI 站点视觉套路。
6. **多级降级容错**：翻译失败回退原文、抓取失败回退 RSS、Redis 未配置静默降级——单点故障不拖垮整站。
7. **可选栏目邮件摘要**：订阅者自由组合 AI / 传媒 / 私募，Vercel Cron + Buttondown 标签实现精确触达。
8. **统一云端数据层**：文章、论文、传媒、私募与订阅偏好全部落入 Upstash Redis，抓取任务与阅读请求解耦。

## 六、作品完成度与现状（诚实说明）

- 外部豆包情报脚本的部分批次仍可能降级，但网站内置 RSS 抓取、Redis 历史数据与本地快照会共同兜底。
- 未配置 Upstash Redis 时不会持久化；首次部署可回退现场抓取，正常生产环境应配置 Redis。
- RSS 源可能限流或临时 403，定时同步会保留上一次成功数据，不会让页面变空。

> 说明这些边界，是为了体现项目对生产约束（限流、依赖可用性、Serverless 限制）有清醒认知，并已通过降级策略兜底。

## 七、未来可拓展方向

- 在三大栏目之上继续增加机构 / 赛道级订阅。
